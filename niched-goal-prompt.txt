# Niched Goal Prompt For Kindario

Use this prompt when implementing the Kindario static front page.

```text
/goal Build the Kindario static front page inside `template/`, using screenshot references as the primary visual authority and the current project planning files as requirements.

Read first:
- `project_brand.md`
- `PLAN.md`
- `design.md`
- `prd.md`
- `roadmap.md`
- `AGENTS.md`

Project context:
- Brand: Kindario
- Repository: https://github.com/eduardosrios/kindario-template
- Niche: charity, NGO, donation, nonprofit, volunteering, volunteers, poverty, hunger, children, Africa, giving, crowdfunding, fundraising, donor trust, Gift Aid, humanitarian relief, volunteer recruitment, social impact, humanitarian organization, philanthropy
- Stack: static HTML, CSS, and JavaScript
- Build folder: `template/`
- Reference folder: `screenshots-references/`
- Selected-reference folder: `used/`

Scope:
- Build only the front page.
- Do not build backend, database, admin panel, CMS/PHP code, authentication, real donation processing, or real form submission.
- Do not use React, Next.js, Vue, Angular, Vite, or a build step unless explicitly requested.
- Do not mention the word "template" in visible site copy.
- Treat Kindario as a real site, but do not invent real-world proof, awards, partners, donation totals, certifications, or outcomes.

Primary visual authority:
- `screenshots-references/` is the primary visual source of truth.
- Do not treat screenshots as loose inspiration only.
- Do not build from a generic charity landing-page skeleton.
- Build from the selected screenshot reference direction, adapted to Kindario.

Before implementation:
1. Use Product Design if available and relevant.
2. Use installed skills for senior UI/UX, senior frontend development, responsive design, and SEO.
3. Visually inspect `screenshots-references/`.
4. Inspect `dont-create-like-this/` if it exists and avoid those patterns.
5. Inspect `imagens/` and any other local asset folders.
6. Select a maximum of 30 references from `screenshots-references/`.
7. Prefer premium references with strong typography, editorial rhythm, powerful image handling, high-trust CTAs, strong footer structure, and polished responsive potential.
8. Avoid references that feel generic, exploitative, overly sentimental, outdated, repetitive, or unsuitable for a humanitarian trust-building site.
9. Copy every selected reference into `used/`.
10. Create or update `used/manifest.md`.

The `used/manifest.md` file must explain:
- Which reference files were selected.
- Why each reference was selected.
- Which parts influence the Kindario implementation.
- How the final design differs enough to avoid copying.

Extract a concrete Kindario design direction before writing production HTML/CSS/JS.

The extracted direction must cover:
- Typography scale and font pairing.
- Editorial section rhythm.
- Layout density.
- Header and navigation treatment.
- Hero composition.
- Donation/support CTA treatment.
- Volunteer CTA treatment.
- Card composition.
- Image cropping and image treatment.
- Icon style.
- Color palette.
- Contrast strategy.
- Spacing system.
- Animation and interaction style.
- Footer structure.
- Responsive behavior patterns.

Expected Kindario art direction:
- Editorial field reports meet a high-trust humanitarian action platform.
- Professional, warm, credible, and modern.
- Strong enough for donors and partners.
- Clear enough for volunteers and first-time visitors.
- Human without being manipulative.
- Premium without looking like a generic charity template.

Front page structure:
- Header/navigation.
- Hero section.
- Exactly 7 main sections after the hero.
- Footer.
- Do not count subsections or columns as additional main sections.

Recommended 7-section content architecture:
1. Trust and credibility framing.
2. Mission and impact model.
3. Campaigns or impact areas.
4. Giving/support pathway without fake payment processing.
5. Field story, report, or documentary-style update.
6. Volunteer and partner pathways.
7. Final story-led CTA.

You may adjust these section themes if the selected references support a stronger page, but the page must remain aligned with the Kindario niche and must still contain exactly 7 post-hero main sections.

Copy requirements:
- English-only.
- Clear and specific.
- No lorem ipsum.
- No fake partner logos.
- No unverifiable statistics framed as fact.
- No exploitative poverty language.
- No guilt manipulation.
- No pretending a static form processes donations.
- Use respectful humanitarian storytelling.
- Make CTAs clear and realistic.

Allowed technology:
- Bootstrap where useful.
- 1 to 3 fonts maximum.
- Font Awesome where useful.
- GSAP only if it materially improves the interaction quality.
- Three.js only if truly appropriate and verifiable.
- Other reliable frontend libraries only when they solve a real need.

Images:
- Use high-quality local images from `imagens/` when suitable.
- Generate or source additional high-quality images only if local assets do not support the selected direction.
- Avoid watermarked images.
- Avoid misleading image/copy pairings.
- Use truthful alt text.
- Avoid exploitative image treatment.

SEO:
- One H1 only.
- Descriptive title tag.
- Meta description.
- Canonical placeholder.
- Open Graph metadata.
- Twitter card metadata.
- JSON-LD for Organization/WebSite/WebPage where appropriate.
- Semantic headings.
- Descriptive alt text.
- No keyword stuffing.

Accessibility:
- Keyboard-accessible controls.
- Visible focus states.
- Semantic landmarks.
- Sufficient contrast.
- Reduced-motion support for heavy animation.
- Descriptive controls and links.

Responsive verification:
- Verify with browser screenshots or Playwright at 375px, 430px, 768px, 1024px, 1366px, and 1440px.
- Confirm no horizontal overflow.
- Confirm no broken local assets.
- Confirm no missing images.
- Confirm no overlapping text.
- Confirm no clipped CTAs.
- Confirm header and mobile navigation work.
- Confirm interactive controls work.
- Confirm no console errors.

Visual QA:
- Compare rendered screenshots against selected references in `used/`.
- Fix any section that looks generic, weak, visually disconnected, or insufficiently informed by the selected references.
- Confirm the final page is original and not a direct copy.

If blocked:
- Stop before implementation if `screenshots-references/` is missing or cannot be inspected.
- Stop if selected references cannot support a premium visual direction.
- Stop if available images are watermarked, low-quality, misleading, or insufficient and no acceptable generated/sourced alternative can be used.
- Stop if verification tools cannot run.
- Explain the blocker, affected requirement, needed user decision, and possible fallback.

Definition of Done:
- `template/` contains a complete static Kindario front page.
- `used/` contains every selected screenshot reference.
- `used/manifest.md` documents selection, influence, and originality.
- The page has header, hero, exactly 7 post-hero main sections, and footer.
- The design visibly follows the selected screenshot-led direction.
- The result looks premium, senior-level, and non-generic.
- The result is respectful to the humanitarian niche.
- The page does not copy any reference 1:1.
- SEO, accessibility, responsive, interaction, and asset checks pass.
- Final response summarizes changed files, selected reference strategy, verification performed, and any GitHub publishing limitation.
```
